Introduction

NERO Character Creator (NCC) is a tool designed to help NERO players plan out the development of their characters. It can also be used by plot members to store NPC stats and by Chapter owners to store backups of their players' characters.

Basic Use

The NERO Character Creator user interface consists of four main tabs: the Info tab, the Build tab, the Experience tab, and the Backstory tab.

Info Tab

The Info tab contains information about the character.

Player NameThe name of the person that plays this character.
Character NameThe character's name.
Date CreatedThe day this character was placed "on the books."
RaceSelecting your race automatically updates the costs of relevant skills. It must be set in order to purchase racial skills.
SubraceFor example, you may be a Lizard Scavenger or a Quentari Elf. This is completely for flavor purposes.
ClassChanging this re-calculates your spent build and current body.
Primary and Secondary SchoolsChanging one automatically changes the other.
Spirit and Body EffectsFormal Magic effects on your spirit and Body. If you don't know what these are, you probably don't have any.
Death HistoryThis automatically calculates the number of black stones you will have on your next death.

Build Tab

On this tab, you manage purchased skills and your spell tree.

Purchasing Skills

Skills can be purchased by selecting them from the drop down menu and clicking "Add."

Certain skills, namely Proficiency and Craftsman Other, have an option that you can choose.  These options are written in the "Note" field.

If the skill has multiple options (like a Proficiency) then you should separate the options with commas.  If it doesn't, just write it out.

Building a spell tree

Your primary and secondary spell tree are defined on the Info tab, but you can update your spell tree here, if you have the prerequisites.  This tree enforces legality, so you can just add a 9th level spell in order to build a 4/4/4/4/4/4/3/2/1 tree.

If you have Enforce Build activated, the tree will revert to its previous state if it would be too expensive to build the tree in order to meet the prerequisite you requested.

Error Messages

You may encounter an error message while attempting to add a skill.  These error messages are fairly self-explanatory.

If you encounter the message "Insufficient options provided" then the program did not understand the options you entered in the way you wrote them.  With proficiencies, note that you must spell out the word "Right" or "Left."

If you encounter a message like this, then you're missing a skill.  Try purchasing that skill before purchasing this one.

If you encounter this message, you are most likely attempting to add a racial skill.  You can set your race on the Info tab.

If you encounter this message, you have the "Enforce Build" option turned on and have attempted to purchase a skill that you cannot afford.

You can disable this by turning that option off.  Alternatively, you can increase your available build on the Experience tab.

You may encounter this error if you attempt to buy Florentine, Two Weapons, or Shield when you already have Style Master, or if you try to buy a weapon skill when you already have Weapon Master.

Experience Tab

This tab is used to add Experience to a character.

The Location and Date buttons are optional - they are used solely if you want to preserve information.

By default, Goblin Blankets are added individually.  If you wish to have them added manually, set the Goblins option to "Group" in the configuration file.

Backstory Tab

The Backstory Tab keeps track of your backstory.  You can use it to track any relevant notes for a character.

Saving and Loading

NERO Character Creator saves files to your My Documents directory by default. This behavior can be changed by modifying the "Save Directory" parameter of the initialization file, ncc.ini.

Certain changes are automatically committed to a file named ncc.yml in the default save directory.

Advanced Use

Configuration

The initialization file (ncc.ini) contains a number of options that can be used to specialize NERO Character Creator for your own use.  These options are case sensitive and may cause the program to crash if they are improperly set.

If you cannot modify this file while it is in the Program directory, copy it and all of the data files into a folder in your personal profile.  Then, modify the shortcut to Start In that folder.  (Right click on the shortcut and click on Properties).

Log Outputthe full filename of the log file.  The default is "nero.log" in the program's directory.  STDOUT can be used if you want the log to be output to the command line.
Log Countthe number of log files that should be saved.  The default is 5.
Log Sizethe maximum size of a log file, in bytes.  The default is 100 KB.
Log ThresholdThis determines how important log messages must be.  The default is 'INFO'.  The options are 'DEBUG', 'INFO', 'WARN', 'ERROR', 'FATAL'.  Set this to 'DEBUG' if you're encountering a recurring bug to report the details.
Skill Count Min WidthDefault is 10.  This controls the size of the increment and decrement buttons for listed skills on the Build page.
Skill Count Max WidthDefault is 20.  This controls the size of the increment and decrement buttons for listed skills on the Build page.
Skill Count Min HeightDefault is 10.  This controls the size of the increment and decrement buttons for listed skills on the Build page.
Skill Count Max HeightDefault is 20.  This controls the size of the increment and decrement buttons for listed skills on the Build page.
Skill EntryChoices are 'Drop Down' or 'Text Box'.  'Drop Down' is default and the 'Text Box' is an auto-completing text box.
TitleThe title of the program.  Any text is acceptable.
Enforce Buildtrue or false.  If true, you cannot add skills or spells if you would then have negative loose build.  The default is false.
Save Directorythe full path of the directory where character sheets will be saved.  The default is the My Documents folder on Windows and the Home directory on other operating systems.  This parameter can be set to "PROGRAM" to instead save in the working directory, which by default is the program's directory.
Skill Datathe name of the skill data file.  The default is 'skills.yml'.
Module 1the filename of a supplemental skill data file.  Default is none.
Module Xthe filename of more supplemental skill data files.  X is a number.
Chapterthe name of the chapter for these characters.
AutosaveThis can be set to True or to False to enable or disable auto-saving, respectively.  The default is True.
GoblinThis can be set to 'Individual' or 'Group'.  If set to Group, the experience from a set of Goblin blankets is calculated without updating your build between them.  The default is Individual.

Creating a Skill Module

A skill module can be easily created by adding skills to a plain text file.  For additional help on any of the definitions declared herein, first refer to the existing module files.

Note that these skill files must follow the YAML standard - namely, this means you must use spaces instead of tabs for indentation.

Example Skill

The following is an example skill, Combat Prowess, which is a prerequisite for "Physical Strike" abilities.

Combat Prowess:
   Cost:
       Fighter: 4
       Scholar: 8
       Rogue: 6
       Templar: 6
   Requires:
       - Weapon

This skill can only be purchased once.  Its costs are defined for each class.  It requires a weapon skill.  That's all you need!

Stacking Skills

If we wanted to create per-day skills based off of this skill, we could do that as follows:

Physical Strike Disarm:
   Cost:
       Fighter: 2
       Scholar: 4
       Rogue: 3
       Templar: 3
   Requires:
       - Combat Prowess
   Max: 0

This skill can be purchased any number of times, as long as you have the skill Combat Prowess.  If you wanted to cap the purchase at some other number, you could do so by setting Max to that number.

Advanced Requirements

Suppose you wanted a skill, Physical Strike Shatter.  To purchase this, you have to have at least one Physical Strike Disarm per purchase.

Physical Strike Shatter:
   Cost:
       Fighter: 3
       Scholar: 6
       Rogue: 4
       Templar: 4
   Requires:
       Physical Strike Disarm: 1
   Max: 0

If you wanted to require 2 Physical Strike Shatters per purchase, you would change the 1 to a 2.

Note that pyramided skills are not yet supported.  If you desire this functionality, please contact the program author.

Racial Skills

It's easy to create skills that can only be purchased by a particular race or that cannot be purchased by a particular race.  The first sort of skill can be created by only having a cost defined for a particular race.  The second sort can be defined by listing that race under 'Prohibits'.

Barbarian Slay:
   Cost:
       Barbarian: 10
Mining:
   Cost:
       Dwarf Fighter: 2
       Dwarf Scholar: 3
       Dwarf Rogue: 3
       Dwarf Templar: 2
   Max: 0
Infusing Celestial Essence:
   Cost:
       Fighter: 8
       Scholar: 4
       Rogue: 6
       Templar: 4
   Prohibits:
       - Biata
       - Barbarian
